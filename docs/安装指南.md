# 环境要求

- **操作系统**：推荐 Ubuntu 20.04+ (aarch64/x86_64) 或其他主流Linux发行版
- **Python版本**：python 3.9~3.11
- **torch_npu版本**：torch_npu 2.6.0
- **CANN版本**：推荐 CANN 8.5.0
- **GCC** >= 9.4.0
- **GLIBC** >=2.27
- **CMAKE** >= 3.18
- **Ninja** >= 1.12

## CANN安装

异构计算架构CANN（Compute Architecture for Neural Networks）是昇腾针对AI场景推出的异构计算架构， 向上支持多种AI框架，包括MindSpore、PyTorch、TensorFlow等；向下服务AI处理器与编程，包含昇腾芯片的底层驱动；发挥承上启下的关键作用，是提升昇腾AI处理器计算效率的关键平台。

目前 Tilelang-Ascend 推荐的 CANN 版本为：

- 昇腾CANN社区版 8.5.0 （发布日期：2026/01/16)

您可以访问[下载链接](https://www.hiascend.com/developer/download/community/result?module=cann&cann=8.5.0)，并根据实际环境选择对应CPU架构 {arch} (aarch64/x86_64) 及 NPU硬件型号 {chip_type} (A3/910b/910/310p/310b) 选择对应的安装包。

推荐下载安装:

| 软件类型 | 软件包说明       | 软件包名称                                                   |
| -------- | ---------------- | ------------------------------------------------------------ |
| Toolkit  | CANN开发套件包   | Ascend-cann-toolkit_8.5.0_linux-**{arch}**.run               |
| kernels  | CANN二进制算子包 | Ascend-cann-kernels-**{chip_type}**_8.5.0_linux-**{arch}**.run |

其中 Toolkit 对于各硬件型号通用；kernels 需要根据型号进行选择。

建议下载 `.run` 后缀文件进行安装，安装脚本为:

```shell
# 更改run包的执行权限
chmod 755 Ascend-cann-toolkit_8.5.0_linux-{arch}.run
chmod 755 Ascend-cann-kernels-{chip_type}_8.5.0_linux-{arch}.run

# 安装到默认路径: /usr/local/Ascend
sudo ./Ascend-cann-toolkit_8.5.0_linux-{arch}.run --install
sudo ./Ascend-cann-kernels-{chip_type}_8.5.0_linux-{arch}.run --install

# 安装成功后设置环境变量
source /usr/local/Ascend/ascend-toolkit/set_env.sh
```

PS：如果用户未指定安装路径，则软件会安装到默认路径下，默认安装路径为:

- root用户: `/usr/local/Ascend`
- 非root用户：`${HOME}/Ascend`

用户也可以通过 `--install-path=<install_path>` 选项指定安装路径。

[社区安装指引](https://www.hiascend.com/document/detail/zh/CANNCommunityEdition/850/softwareinst/instg/instg_0000.html?Mode=PmIns&InstallType=netconda&OS=openEuler) 提供了详细的安装流程说明与依赖配置选项, 用户可进一步对 CANN 进行配置。

# 通过源码安装 Tilelang-Ascend

TileLang-Ascend 当前仅支持通过源代码编译安装。

## 依赖配置

### 系统库依赖

`apt` 源安装方式为:

```shell
sudo apt-get update
sudo apt-get install -y clang lld cmake make libzstd-dev zlib1g-dev patch patchelf
```

`yum` 源安装方式为:

```shell
sudo yum install -y clang lld cmake make libzstd-devel zlib-devel patch patchelf
```

### python库依赖

推荐使用 conda 对 python 环境进行管理，在激活 conda 环境后运行：

```shell
pip install setuptools wheel packaging cmake pybind11 ninja
```

随后安装 `torch_npu` ：

```shell
pip install torch_npu==2.6.0
```

如果出现报错`ERROR: No matching distribution found for torch==2.6.0+cpu`，可以尝试手动安装torch后再安装 `torch_npu`：

```shell
pip install torch==2.6.0+cpu --index-url https://download.pytorch.org/whl/cpu

pip install torch_npu==2.6.0
```

## 安装流程

### 1. 克隆 Tilelang-Ascend 源码

```shell
git clone https://github.com/tile-ai/tilelang-ascend.git --recursive -b npuir
```

此命令会递归克隆 `3rdparty` 中的仓库, 所需时间较长, 请耐心等待。

### 2. 构建 Tilelang-Ascend

TileLang-Ascend 提供了一键安装脚本 `install_npuir.sh`，可自动完成 `AscendNPU-IR` 和 TileLang 的编译构建。在 `tilelang-ascend` 目录中运行：

```shell
bash install_npuir.sh
```

- 注：脚本运行时会自动安装 `requirements.txt` ，推荐在激活的 conda 环境中运行上述脚本。

随后将构建好的 `Tilelang-Ascend` 目录添加到 `PYTHONPATH` :

```shell
export PYTHONPATH=/<path>/tilelang-ascend:$PYTHONPATH
```

**构建选项**

- `--enable-llvm` ：使用llvm
- `--bishengir-path=<path>`  ：\<path> 为 `bishengir-compile` 所在目录, 通过此选项进行指定。

## 运行验证

```shell
conda activate <tilelang_env>

# 设置CANN环境变量（以root用户默认安装路径`/usr/local/Ascend`为例）
source /usr/local/Ascend/ascend-toolkit/set_env.sh

# 设置tilelang路径
export PYTHONPATH=/<tilelang_path>/tilelang-ascend:$PYTHONPATH

# 在tilelang-ascend 目录下
cd examples
python vec_add_1d.py
```

成功运行并输出如下结果：

```
Reference result (PyTorch):
tensor([0.8329, 1.0024, 1.3639,  ..., 1.0796, 1.0406, 1.5811], device='npu:0')
TileLang kernel result:
tensor([0.8329, 1.0024, 1.3639,  ..., 1.0796, 1.0406, 1.5811], device='npu:0')
```

表明安装已成功完成。
